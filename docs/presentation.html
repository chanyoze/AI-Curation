Why

AI·테크 뉴스는 너무 빠르게, 너무 많이 쏟아진다

🌊 정보 과부하

외신 AI 뉴스가 매일 수십 건. 원문은 영어라 따라가기 어렵고, 무엇이 중요한지 가려내기 힘들다.

🔎 키워드 검색의 한계

"요즘 LLM 추론 비용 얘기 뭐 있었지?" 같은 맥락 질문은 단순 키워드 검색으로 답이 안 나온다.

⏰ 놓치는 속보

중요한 기사가 올라와도 직접 들어가 보기 전엔 모른다. 푸시받을 방법이 없다.

→ AI로 AI 뉴스를 큐레이션한다. 수집·요약은 자동, 검색은 자연어, 알림은 실시간.

What

핵심 기능 4가지

📡

자동 수집

The Guardian API에서 AI·테크 기사를 스케줄러로 주기 수집. URL/ID 해시로 중복 제거.

🧠

LLM 가공

영문 기사를 한국어로 번역·요약·태깅. 기사당 단 2번의 LLM 호출로 처리.

💬

RAG 검색

자연어 질문 → pgvector 유사도 Top-K → 근거 기사와 함께 답변 생성.

🔔

실시간 알림

새 기사 수집 시 SSE로 구독자에게 즉시 푸시 + 캐시 자동 갱신.

데모 시나리오: "최근 오픈소스 LLM 동향 알려줘" → 요약 답변 + 출처 3건 + 새 기사 토스트

✅ 핵심 설계

전문 번역 안 함 — 영어를 한국어로 요약하면 그게 곧 번역+요약. 출력 토큰을 아끼고 원문은 그대로 보관 + 출처 링크.

❌ 피한 안티패턴

번역·요약·태깅·임베딩을 4번 따로 호출 / 전문을 통째로 번역. → 무료 쿼터를 빠르게 소진하는 길.

왜 SSE인가

새 기사 알림은 서버→클라 단방향. WebSocket의 양방향 오버헤드가 필요 없다.

학습 포인트 (스트레치)

EventSource는 커스텀 헤더 불가 → JWT를 쿼리파라미터/쿠키로 넘겨 연결 시점에 검증. 일반 HTTP 인증과 다른 타이밍.
